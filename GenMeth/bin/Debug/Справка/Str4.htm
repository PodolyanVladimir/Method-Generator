Коллекция свойств

После полной комплектации коллекции имён можно приступить к работе с коллекцией свойств. Для этого служит пункт меню "Проект\Редактор свойств".

Если коллекция имён недоукомплектована, например, ни для всех таблиц заданы столбцы, приложение не позволит войти в "Редактор свойств".

После входа в редактор свойств, в списке будет выведен список столбцов. Для определения или изменения свойств столбцов требуется выделить соответствующую строку списка и вызвать диалоговое окно, кликнув по кнопке "Редактировать свойства", расположенной вверху окна, или воспользоваться контекстным меню, кликнув правой кнопкой мыши по списку столбцов.

Определение свойств подразумевает выбор одного из трёх значений: "Первичный ключ", "Внешний ключ" и "Столбец данных".

При использовании технологии индексирования ключей Guid дополнительных действий для первичных и внешних ключей не требуется.

При использовании технологии индексирования ключей AutoIncrement необходимо указать начальное значение и шаг инкремента для первичного ключа. Для внешнего ключа дополнительных действий не требуется.

Для категории свойств "Столбец данных" должен быть выбран тип данных. В списке свойств поддерживаются следующие типы: String, Boolean, Byte, Char, DateTime, Decimal, Double, Int16, Int32, Int64, SByte, Single, TimeSpan, UInt16, UInt32, UInt64.

Для любого из типов можно задать выражение. Ответственность за корректность вводимого выражения лежит на пользователе.

Для типа String требуется установить значение максимальной длины строки.

По окончании работы с редактором свойств, из него следует выйти. При этом приложение предложит сохранить изменения. Сохранение изменений при выходе из редактора свойств происходит не в файл, поэтому для окончательного сохранения данных решения требуется выполнить файловое сохранение в главном окне приложения.
